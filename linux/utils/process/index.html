Process Management CLI Utils

Published at: October 2, 2023 | Last modification: March 16, 2024

pgrep/ pwait / pkill from procps-ng

These 3 tools support rich condition search on processes.

pwait/pkill by pid is not supported.

pgrep

pgrep always prints a delimiter-separated (configurable by -d) list of pid.

  • Selectors are always logically intersected, i.e. all must match.
  • List supplied to a single selector are always unioned.
  • Use -v to negate the whole selector combination.
1# select processes whose EUID=1000, AND processes whose cmd name is exactly 'code'
2pgrep -u 1000 -x code

ps from procps-ng

ps suffers from the curse of compatibility for multiple historical implementations. Some effort exists, but there seems to be no wide-spreading replacement for it by 2022.

Normally, kernel doesn’t ship process visibility control, i.e. everyone can see everyone else’s process via /proc.

Without any args, ps mimics the vanilla BSD ps.

selection

If multiple selector are used, the result is union-ed.

1# select processes whose EUID=1000 
2[cedo@juejueko] ~ $ ps U 1000 | wc
3    152    1735   42120
4
5# select processes whose EUID=1000, OR processes whose cmd name is exactly 'code'
6# which, is not usually people want.
7[cedo@juejueko] ~ $ ps U 1000 -C code | wc
8    152    1737   42128

ps alone can’t achieve intersection for selectors. If that’s desired, use pgrep to get a list of pid first.

Use -N or --deselect to negate the selection.

A list can be either comma-separated single shell token, or whitespace-separated single or multiple shell token(s).

  • words (user name / login / pid, etc.) in a list are always logically union-ed.

  • whitespace one triggers more verbose output as an effort to solve dirty compatibility problems.

    1# ps -p 1,4839
2    PID TTY          TIME CMD
3      1 ?        00:00:00 systemd
4   4839 ?        00:00:00 code
5# ps -p 1 4839
6    PID TTY      STAT   TIME COMMAND
7      1 ?        Ss     0:00 /sbin/init
8   4839 ?        Sl     0:00 /opt/VSCode-linux-x64/code --ms-enable-electron-run-as-node /opt/VSCode-linux-x64/resources/app/out/bootstrap-fork --type=watcherServiceParcelSharedProcess

select all:

1ps -A
2# identical to
3ps -e

By pid(s):

1# only 1 arg allowed
2ps 1122
3ps -1122
4
5# pid list union
6ps p 1122,1
7ps -p 1122,1
8ps --pid 1122,1

By parent pid(s):

1# the only syntax
2ps --ppid 1,6877

By user(s):

1# by EUID or login (name), comma-separated list
2ps U root,1000
3ps -u root,1000
4ps --user root,1000
5
6# by RUID or login (name), comma-separated list
7ps -U  root,1000
8ps --User root,1000

By cmd name(s):

cmd name is /proc/*/cmdline truncated to a simple name.

1# comma-separated list
2ps -C code,chrome

Notable flags:

  • Without any flags: PID TTY TIME CMD

  • ps -f: documented as “full-format”, but actually only UID PID PPID C STIME TTY TIME CMD

  • ps -F: documented as “extra full-format”, but actually -f plus SZ RSS PSR
