Java has a built-in security infra-structure since the applet era. Back in those days the client’s JVM running applet needs to receive bytecode .class file from server, thus a security mechanism is crucial.

The core of this infrastructure is the java.lang.SecurityManager class. A unique JVM-wide global instance of this class can be “installed” by System.setSecurityManager(), and retrieved by System.getSecurityManager(). By default no instance of SecurityManager is installed, and the getter returns null.
