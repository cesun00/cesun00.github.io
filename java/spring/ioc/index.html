interface StartupStep and ApplicationStartup

Spring IOC 5.3 introduces a minor new feature of measuring container startup metrics, and work as follows

  1. Logically, ApplicationContext’s initialization is divided into steps. Such division is arbitrary, and is up to the design choice of the Spring team.
  2. ApplicationContext now holds an implementation of ApplicationStartup applicationStartup.
  3. Each logical step begins and ends with a pair of calls:
    • start by applicationStartup.start("step name"), which returns an StartupStep instance step
    • end by step.end()

Implementations of StartupStep and ApplicationStartup are free to decide what happen when these 2 APIs are called. The default implementation of ApplicationStartup composited in all ApplicationContext, as well as its returning StartupStep, is no-op for these 2 calls, i.e. does not measure by default, thus no performance punishment. Call ConfigurableApplicationContext#setApplicationStartup to change the default.
