Data From Virtualized List

Published at: February 1, 2024 | Last modification: May 6, 2024

js node playwright ts

Modern web design may choose to use virtual list or virtual table when displaying large amount of structured data. The central idea is to increase page rendering performance by avoid creating each node for each data item. Virtualized list keeps a few DOM nodes (most commonly <li>) only enough to fill its viewport, and change the content of those DOM nodes by inspecting location of slide of the scrollbar when scroll event is fired.

In order to make the scrollbar display correct ratio despite the fact that there is only a few <li> nodes, virtualized list usually takes the following DOM structure:

 1    <div style="height: 200px; overflow: scroll;">
 2        <!-- a javascript computed height, to make the scrollbar ratio render correctly -->
 3        <div style="height: 20000;">
 4            <!-- but there are actually a few li enough to fill the 200px viewport -->
 5            <li></li>
 6            <li></li>
 7            <li></li>
 8            <li></li>
 9            <li></li>
10            <li></li>
11            <li></li>
12            <li></li>
13        </div>
14    </div>

The use of virtual list introduce difficulties for frameworks like playwright or puppeteer.

solution 1

1            const locator = page.locator("li[role='option']", {hasText: targetAddr});
2            while (await locator.count() != 1) {
3                await page.keyboard.down("ArrowDown");
4            }
5            await locator.click();

solution 2

1            const scroller = page.locator('ul[role="listbox"] > div');
2            const locator = scroller.locator("li[role='option']", {hasText: targetAddr})
3            while (await locator.count() != 1) {
4                // be careful about this scroll step length
5                await scroller.evaluate((div) => div.scrollBy(0, 30));
6            }
7            await locator.click();

solution 3

 1            const scroller = page.locator('ul[role="listbox"] > div');
 2            await scroller.evaluate(async (div, addr) => {
 3                while (true) {
 4                    const it = document.evaluate(`.//span[text()="${addr}"]`, div).iterateNext();
 5                    if (it != null) {
 6                        (it as HTMLElement).click()
 7                        return;
 8                    }
 9                    div.scrollBy(0, 10);
10                }
11            }, targetAddr);

Experienced Web programmer will notice the problem in the code above.

 1            const scroller = page.locator('ul[role="listbox"] > div');
 2            await scroller.evaluate(async (div, addr) => {
 3                while (true) {
 4                    const it = document.evaluate(`.//span[text()="${addr}"]`, div).iterateNext();
 5                    if (it != null) {
 6                        (it as HTMLElement).click()
 7                        return;
 8                    }
 9                    div.scrollBy(0, 10);
10                    // given a chance for the scroll event handler of to do its stuff
11                    // https://stackoverflow.com/questions/779379/why-is-settimeoutfn-0-sometimes-useful
12                    await new Promise(resolve => setTimeout(resolve, 0));
13                }
14            }, targetAddr);
